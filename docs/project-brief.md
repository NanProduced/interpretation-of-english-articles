# 项目简述

## 产品目标

打造一个面向非零基础英语用户的轻量微信小程序，以结构化 AI Workflow 为核心能力，帮助用户真正读懂英文文章，而不是只得到一段普通翻译。

## 核心价值

- 帮助用户理解真实英文文章
- 输出稳定、结构化结果，而不是聊天式回答
- 在英文阅读场景下提供比通用大模型更好的使用体验

## 主要用户

- 在校大学生
- 备考用户
- 职场人士
- 英语阅读爱好者

## 核心流程

1. 用户提交文章文本。
2. 先进行内容安全审核。
3. 预处理模块清理格式并检测问题。
4. 后端 Workflow 执行结构化分析。
5. 前端渲染分层解读结果。

## 目标解读维度

- 词汇标注
- 语法分析
- 长难句拆解
- 逐句翻译
- 全文翻译
- 后续可选篇章分析

## 产品边界

本项目是：

- 聚焦阅读理解的工具型产品
- 由 Workflow 驱动的结构化解读产品
- 打开即用、低摩擦的小程序

本项目不是：

- 面向零基础用户的英语启蒙课程
- 通用 AI 聊天产品
- 社交产品
- 内容平台

## 架构方向

- 前端：Taro + React + TypeScript
- 后端：FastAPI + LangGraph + PydanticAI
- 安全审核：微信 `msgSecCheck`
- 数据存储：CloudBase 或同类托管服务
- 未来富化：等基础 Workflow 质量稳定后再引入 RAG

## 核心工程原则

- 先定义 Schema，再扩功能
- AI 负责语境相关解释，不负责确定性词典事实
- 能用规则或查表完成的富化，尽量不走 LLM
- 在早期阶段，输出稳定性比功能广度更重要

## 当前主要风险

- 输出质量和稳定性还没有建立完整评估体系
- 长文章延迟可能影响小程序体验
- span 对齐和结果页分层渲染会比 prompt 本身更难
- 合规材料和审核链路需要尽早准备
