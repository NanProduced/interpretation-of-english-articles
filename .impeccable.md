# Design Context

## Users

目标用户为有一定英语基础的成年人，包括在校大学生、职场人士、备考用户和英语阅读爱好者。

用户使用场景高度碎片化——可能在通勤路上、图书馆、通勤途中，每次使用时长不确定，因此产品必须：

- 极低认知负担：用户"带文章来就能用"，无需复杂配置
- 分层信息架构：默认层干净，高阶信息按需展开
- 快速返回结果：解读即点即现，不等待加载

## Brand Personality

**简约但细节满满。轻盈且专业。**

参考 Goodnotes 的设计调性：干净纸张质感、精致的标注呈现、无多余的视觉元素。

产品不是"英语学习 app"，而是"专业的文章解读工具"——克制、专业、有质感。

## Aesthetic Direction

### 参考：Goodnotes

Goodnotes 的核心设计特质：

1. **纸张质感**：背景非纯白，带轻微暖灰或米白，有"真实纸张"的错觉
2. **标注自然**：高亮、下划线等标注像真实文具，而非 UI 控件
3. **留白充足**：内容区大气，呼吸感强，不拥挤
4. **细节精致**：图标圆润、间距一致、字体优雅，连 tab 指示器都打磨过
5. **动效轻盈**：轻微的弹性动画，按压反馈自然，不花哨

### 视觉关键词

- **纸感**：背景可用 `#FAF9F9` 或 `#F8F8F8` 替代纯白，带微暖色调
- **留白**：内容区 `32rpx` 边距，段落间 `32rpx`，句间 `16rpx`
- **圆润**：图标、按钮、卡片均使用 `24rpx` 起的小圆角
- **墨水感**：标注颜色像真实荧光笔（半透明 background + 无边框），而非按钮

### 抗参考（绝对不要）

- 老式 HTML 表格/列表式布局
- 大量 box-shadow 滥用
- 纯白背景 + 大量 border 分隔线
- 移动端狭小空间内堆砌信息
- 过度使用深色分割线

### 主题

白底为主，品牌色 `#030213` 作为主文字和重点元素色。

## Design Principles

### 1. 纸质感优先

背景和排版模拟真实阅读体验，而非 web 控件堆砌。

- 背景带微暖色调（`#FAF9F9` 或类似）
- 正文和句子保持舒适的行高（`1.8`）
- 标注使用半透明背景色，模拟荧光笔

### 2. 分层而非堆砌

信息分层展示，屏幕永远不拥挤：

- **默认层**：英文正文 + 重点词高亮，翻译隐藏
- **查询层**：点击单词弹出查词（WordPopup）
- **深度层**：句尾 chip/段间卡片，按需展开
- **精读层**：全部标注和入口，适合深入学习

同一屏内严禁同时展示所有信息。

### 3. 标注即文具

标注组件（InlineMark）模拟真实文具，非 UI 控件：

- 高亮：半透明背景色 + 4rpx 圆角，无边框
- 下划线：细虚线，颜色继承语义色
- 禁止：高饱和度背景、粗边框、过度阴影

### 4. 移动端空间意识

手机屏幕是狭小空间，每个元素必须"值得存在"：

- 边距和间距统一使用 `rpx`，基准 `8rpx` 网格
- 信息密度低：宁可留白，不要压缩
- 交互热区足够大（最小 `44rpx`）

### 5. 精致细节

简约不等于简陋，每个细节都要打磨：

- 字体：系统字体栈，等宽字符用 `'SF Mono', Menlo`
- 图标：圆润风格，与圆角系统一致
- 动效：轻微弹性（`ease-out` 200ms），按压反馈自然
- 过渡：页面/面板切换有方向感，不生硬

## 微信小程序特定约束

本项目使用 **Taro 3 + React** 开发微信小程序。以下是必须遵守的平台约束和最佳实践。

### 强制约束（踩坑整理）

#### 样式限制

| 约束 | 说明 | 应对 |
|------|------|------|
| **不能用 id 选择器** | `#id` 和 `[attr]` 选择器在组件内无效 | 全部用 class 选择器 |
| **后代选择器慎用** | `.a .b` 在某些极端情况表现异常 | 尽量使用直接子选择 `.a > .b` |
| **标签名选择器慎用** | `button { }` 在组件内可能不生效 | 用 class 替代 |
| **样式隔离** | 组件默认样式隔离，app.wxss 不影响组件内部 | 使用 `externalClasses` 或 `addGlobalClass` |
| **单位** | WXSS 只支持 `rpx` 和 `px`（不建议用 `rem`/`em`） | 统一用 `rpx` |

#### 原生组件层级

微信小程序有一类**原生组件**（如 `video`、`map`、`canvas`、`camera`），它们的渲染层级高于普通 DOM，可能遮挡弹窗。

- **不要用 `z-index` 尝试覆盖原生组件**，在真机上无效
- 覆盖原生组件只能用 `cover-view` / `cover-image`
- `cover-view` 功能受限：不支持复杂布局（flex 有限）、字体截断问题（iOS）
- **建议**：本项目核心页面（结果页）不使用 `video`、`map` 等原生组件，避免层级问题

#### 交互事件

| 问题 | 说明 |
|------|------|
| **onClick vs onTap** | Taro/React 中两个事件都支持，但混用可能导致行为不一致。建议统一使用 `onClick`（Taro 会转换为小程序 `bindtap`） |
| **冒泡处理** | 微信小程序事件默认冒泡，与 Web 行为不同。注意在子元素上使用 `catchClick`（阻止冒泡） |
| **输入框** | `input` 组件在滚动区域内可能被系统键盘抬起遮挡，需用 `scroll-into-view` 或 `adjust-position` 处理 |

#### setData 性能

小程序通过 `setData` 传递数据给渲染层，数据过大会影响性能：

- **避免频繁 setData**：每次 setData 都会进行一次数据序列化传输
- **避免传递大数据**：不要把整页数据都放在一个 setData 调用里
- **局部更新**：只更新变化的最小数据单元

#### 资源限制

| 限制 | 数值 | 应对 |
|------|------|------|
| 代码包大小 | 单包 ≤ 2MB，分包总和 ≤ 20MB | 及时清理未用代码和资源 |
| 资源文件 | 图片/音频总和 ≤ 200KB（超出会影响加载） | 大资源托管服务器 |

### 导航与页面结构

#### TabBar

- **首页可用原生 TabBar**，但自定义 TabBar 更灵活
- Tab 数量：`2-5 个`，建议不超过 `4 个`
- 本项目使用自定义 TabBar（`pages/tabbar/`）

#### 导航栏

- 原生导航栏高度：`44px + statusBarHeight`（iOS）或动态计算
- 可在 `app.config.ts` 中配置 `navigationBarTextStyle: 'black'/'white'`
- 自定义 NavBar 时需计算 `statusBarHeight` 和 `navigationBarHeight`
- **注意**：iOS 微信支持右滑返回，安卓用物理返回键

#### 页面路径

- 页面路径在 `app.config.ts` 的 `pages` 数组中声明
- 页面栈最多 `10 层`，避免过深导航

### 最佳实践

#### 1. 面板交互

- **底部弹出面板**（WordPopup、BottomSheetDetail）：用 `position: fixed` + `align-items: flex-end`，从底部滑入
- 面板背景遮罩：`rgba(0, 0, 0, 0.4)`，点击遮罩关闭
- 面板圆角：`24rpx` 起，与卡片系统统一
- 面板最大高度：`70vh` / `80vh`，内容可滚动

#### 2. 加载状态

- **局部加载优先**：只在触发区域显示加载反馈，而非全屏 loading
- 使用 `ScrollView` 的 `refresher` 属性实现下拉刷新
- 加载动画：简洁圆周/脉冲动画，1.5s 后自动提示，超长等待提供取消按钮
- **禁止**：同时出现多个加载动画

#### 3. 空状态与错误

- 异常状态必须有明确提示和可返回的路径
- 表单错误：在顶部标识错误原因，定位到错误字段
- **禁止**：用户遇到异常后无处可去

#### 4. 字体规范（微信官方）

微信小程序字体与系统一致，推荐字号：

| 场景 | 字号 |
|------|------|
| 导航标题 | 17pt |
| 正文 | 15pt |
| 二级正文/说明 | 14pt |
| 辅助信息 | 12pt |

微信官方的色板：

| 用途 | 颜色 |
|------|------|
| 主文字 | `#000000` |
| 次级文字 | `#576b95` |
| 页面背景 | `#efeff4` |
| 分割线 | `#e5e5e5` |

#### 5. 点击热区

微信官方建议点击区域物理尺寸 `7-9mm`，对应 rpx 约 `56-72rpx`。

- **最小热区**：`44rpx × 44rpx`
- **推荐热区**：`56rpx` 及以上
- Chip/Button 等可点击元素，确保足够大

#### 6. safe-area 适配

iPhone X 及以上有底部安全区，组件需适配：

```scss
padding-bottom: constant(safe-area-inset-bottom);
padding-bottom: env(safe-area-inset-bottom);
```

#### 7. 分包加载

代码包超限时使用分包：

- 主包：TabBar 页面、核心流程页面
- 分包：次级页面（如设置、历史记录）
- 分包预加载：预知用户行为时提前加载
